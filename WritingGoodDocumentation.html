Students often think that the most important part in leaning Computer science is coding and solving problems. Therefore, most students only focus on the code and ignore everything else. Although they can solve the problems, the code is difficult to maintain and for other people to follow. In that case, documentation is a the solution.

What is Documentation?

Documentation is a text go along with computer software to illustrate how to use the code, or explain the meaning of code.

Why Documentation?

  • Knowledge Preservation: ensure effective knowledge transfer.
  • Quality increasing: make project well-organized.
  • Make your project consistent overtime.

How to write Documentation?

  • Determine what information should be included
  • Determine where to locate your documentation
  • Choose appropriate tools based on your programming language.
  • Know who will read your documentation

Tips for writing good documentation

  • Visualize your documentation
  • Offer examples in your documentation
  • Make documentation easy to understand
  • Figure out impediments
  • Test your documentation
  • Write documentation for your code by yourself
  • Refer to different sources about documentation for getting better

Try it yourself

  1. Resources:
    • Here is a good book available in pdf about documentation.
    • Go to lammps.sandia.gov/ to see a good example of writing documentation
  2. Writing documentation:
    From now, you can start to write documentation for yourself based on what you learnt so far with any of the following tools: RoboHelp, Help and Manual, Doc-To-Help, MadCap Flare, or HelpLogix. You also could use any other tools that you prefer. Look carefully at lammps.sandia.gov/ and try to documentation with your own code. Good luck!
