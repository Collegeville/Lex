Software Debugging

  1. Error, fault, failure:
    • Error: misunderstanding or misconceptions of programmers
    • Fault (also called bug): The incorrect code
    • Failure: The observable incorrect behavior of the software
    • Error causes fault, fault causes failure.
  2. Debugging approaches: Once we have failure, we need some solutions:
    • Testing: test some input domain and compare to the expected output domain
    • Static verification: similar to testing but you have to test all imput domain/li>
    • Inspection: Look at the code again to find mistake<
    • Formal proof of correctness: proof to verify the expected result from the program

Software Testing:

  1. Overview:
    • Testing is an attempt to find mistake in the code
    • Testing successfully can avoids the system failure, which may lead to some serious disasters
  2. Black-box testing: focus on system behavior, and do not deal with the code, often go along with boundary testing
  3. White-box testing: focus on internal structure of the system, often go along with path testing.

Software Refactoring:

  1. Why refactoring?
    • Refactoring is a process of rewrite your program
    • Refactoring can make the code more understandable, reduces unnecessary code, add more codes, and alleviate the maintenance process
  2. When refactoring?
    • Duplicate code
    • Long method or long class
    • Methods are not in suitable class
    • Lack of necessary comment
  3. How refactoring?
    • Method invocation
    • Inheritance
    • Delegation
