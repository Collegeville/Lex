Introduction to Kanban

Kanban is a tool for managing work flow, which is helpful on optimal flow of work within process. Kanban can be used by anyone from architect, sale teams to students. There are 3 ways for developing Kanban:
  • Visualize work flow
  • Limit work in process
  • Measure and improve work flow

Why Kanban?

  • Kanban help developer teams flexible in planning because they only focus on active work and ignore outsides changes.
  • Kanban minimize the cycle time, bacuase all teams are responsible to keep project go smoothly.
  • Kanban is efficiency because it limits work in process.
  • Kanban visual all metric, which make the team easier to catch up work flow.
  • Kanban allow continuous delivery, which deliver the team's product faster into market.

Exercises:

  1. Exercise 1: Bryant Electronics produces short runs of battery-powered pocket lanterns. You have been asked to reduce inventory by introducing a kanban system. After several hours of analysis you have developed the following data for connectors used in one work cell. How many kanbans do you need for this connector? Provided the following information: Daily demand: 1500 units, Production lead-time: 1 day, Safety stock: 1 day, Kanban size: 250 units.
    (Answer: 12)
  2. Exercise 2: Perkins Lighting wishes to employ a kanban in their new floor lamp production system. Find the size of the kanban and the number of kanbans required. For the floor lamp base, they have provided the following information: Daily demand: 300 units, Holding cost $20/unit/year, Order cost: $10/order, Lead time: 2 days, Safety stock: 600 units.
    (Answer: Size kanban: 300; Number of Kanban: 4)
